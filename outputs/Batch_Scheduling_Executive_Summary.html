Refinery Batch Scheduling Executive Summary

1. 목적

본 시스템은 정유 공장 가솔린 배합 운영에서 판매계획, 제품 스펙, 탱크 heel, 반제품 재고, deadstock, QC 시간을 동시에 고려하여 현실적인 배합 스케줄을 빠르게 생성하기 위한 의사결정 지원 도구입니다.

핵심 목표는 실무에서 설명 가능하고 빠르게 재계산 가능한 계획을 제공하는 것입니다.

2. 현재 운영 문제

  • 판매 주문별 due date와 제품 spec이 다름
  • 제품 하나를 2개 탱크와 6개 배치로 분할 생산
  • 한 배치는 4시간 배합 후 8시간 QC 필요
  • 탱크에는 deadstock가 남아 다음 품질에 영향
  • 반제품 재고와 rundown 제약이 존재
  • 수출 지연 시 demurrage, 내수 지연 시 강한 패널티 발생

3. 채택한 접근 방식

  1. Master MILP: 주문 시작 시점, 탱크 조합, 반제품 투입량 결정
  2. Local Tank Resequencing: carryover contamination 감소
  3. Batch Timing Heuristic: 4h 배합 + 8h QC + 블렌더 최소 gap 반영
  4. Deadstock-aware Refinement: heel을 반영한 배치별 성상 재계산
  5. Inventory-aware Repair: 재고 부족 발생 시 캠페인 지연을 허용하며 inventory 0 이상 유지 방향으로 보정

4. 현재 기본 예시 시나리오

5. 현재 화면이 제공하는 것

  • Planning 탭: Input Center, General Settings, Sales Orders, Blendstocks, Product Tanks
  • Excel 입력: Settings / Orders / Blendstocks / Tanks를 한 파일로 업로드
  • Campaign Gantt: 블렌더 라인별 배치 실행시간 표시
  • Batch Detail: Batch Info, Blend Recipe, Heel/Blend/Target 품질 비교
  • Tank Level / Inventory: 탱크 수위와 반제품 재고 추이
  • Runtime Profile: MILP, resequencing, deadstock postprocess 시간 분해

6. 성능과 운영적 의미

  • 현재 2개월 예시 기준으로 약 30초 수준에서 재계산 가능
  • 재고 shortage를 0 미만으로 두지 않는 방향의 repair 적용
  • 정확한 전역 최적해보다 운영 가능성, 해석성, 응답 속도에 강점
  • 계획 변경 시 대안 시나리오를 빠르게 비교하는 데 적합

7. 실무 활용 가치

  • 판매계획 변경 시 빠른 재시뮬레이션
  • deadstock와 품질 리스크의 정량적 해석
  • 탱크 재사용 전략과 contamination 리스크 비교
  • 간트, 재고, 탱크 수위, 배치 품질 비교를 한 번에 제공

8. 향후 확장 방향

  • inventory overflow repair 추가
  • scenario library와 결과 비교 기능 확장
  • berth/vessel scheduling, tank cleaning 비용 모델 연계
  • 실제 정유사 master data와 연동
