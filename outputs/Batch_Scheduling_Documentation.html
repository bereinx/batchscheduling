Refinery Batch Scheduling Documentation

1. 프로젝트 개요

본 프로젝트는 정유 공장 가솔린 배합 스케줄링 문제를 대상으로 한 Streamlit 기반 의사결정 지원 도구입니다. 판매 주문의 납기, 제품 스펙, 제품 탱크 heel/deadstock, 반제품 재고, QC 대기시간, 블렌더 사용 제약을 함께 고려하여 실무형 배합 계획을 빠르게 생성하도록 설계되었습니다.

현재 구현은 MILP + 로컬 재시퀀싱 + deadstock-aware 후처리 + inventory repair 구조를 사용합니다. 정확한 전역 MINLP보다 설명 가능성과 계산 속도를 우선한 구조이며, 현재 2개월 기본 예시 기준으로 1분 이내 풀이를 목표로 운영됩니다.

3. 문제 정의

  • 판매 제품은 DOMESTIC_LS, EXPORT_LS, EXPORT_HS 서비스로 운영
  • 제품 spec은 RON, RVP, Density, Sulfur, Olefin으로 정의
  • 제품 하나는 일반적으로 2개 탱크에서 총 6개 배치로 생산
  • 배치 1회는 기본 4시간 배합 후 8시간 QC 대기
  • 같은 탱크의 다음 배치는 이전 배치 QC 종료 후에만 가능
  • 같은 블렌더에서는 다른 제품이라도 배치 간 최소 2시간 gap 유지
  • 탱크에는 deadstock가 남으며, 현재 기본값은 탱크 용량의 20%
  • 수출 지연 시 demurrage가 발생하며, 내수는 강한 납기 패널티를 적용

5. 알고리즘 구조

  1. Input Sanitization: 입력값 형식 정리, 숫자/문자 검증, due date 정규화
  2. Master MILP: 주문 시작 시점, 서비스 라인, 탱크 조합, 배치별 반제품 투입량 결정
  3. Local Tank Resequencing: carryover contamination 관점에서 탱크 사용 순서 보정
  4. Batch Timing Heuristic: 4시간 배합, 8시간 QC, 2시간 blender gap 재배치
  5. Deadstock-aware Refinement: heel을 반영한 배치별 성상/배합 재계산
  6. Inventory-aware Repair: 재고가 음수가 되는 구간이 있으면 해당 성분을 많이 쓰는 캠페인을 뒤로 밀어 재계산

5.1 마스터 MILP

마스터 MILP는 다음 의사결정을 포함합니다.

  • 주문 시작 slot 선택
  • 주문별 제품 탱크 사용 여부
  • 배치별 반제품 투입량
  • 납기 지연, 품질 slack, 재고 slack, 탱크 용량 slack

목적함수는 다음 비용의 합을 최소화합니다.

  • 반제품 사용 비용
  • 수출 지연에 대한 demurrage
  • 내수 지연 패널티
  • 품질 spec slack 패널티
  • 반제품 재고 slack 패널티
  • 탱크 용량 slack 패널티

5.2 Deadstock / Heel 처리

heel은 “직전 배치까지 끝난 뒤 탱크에 남아 있는 잔량 및 성상”으로 정의합니다.

  • 첫 배치 heel: 캠페인 시작 전 초기 탱크 상태
  • 두 번째 배치 heel: 첫 번째 배치 종료 후 탱크 잔량/성상
  • 세 번째 배치 heel: 두 번째 배치 종료 후 탱크 잔량/성상

따라서 배치 상세 화면의 HEEL 값은 배치마다 달라질 수 있으며, 실제 탱크 내 잔류물 누적 효과를 표현합니다.

5.3 Inventory-aware 처리

  • 반제품은 horizon 총량뿐 아니라 시간 순서상으로도 해석되도록 후처리
  • 특정 시점 재고가 음수가 되는 경우, 해당 성분을 많이 소비하는 캠페인을 뒤로 미뤄 repair 수행
  • 수요 지연은 허용하되, 재고가 0 미만으로 내려가지 않도록 하는 방향을 우선

6. 현재 사용자 화면

6.1 사이드바

  • 입력값 불러오기: Excel 파일 업로드
  • 최적화 실행: 현재 입력값 기준 모델 실행
  • Final Result: 목적함수, 비용, runtime profile 요약

6.2 Planning 탭

  • Input Center: 현재 입력 요약, 설정 테이블, 템플릿 다운로드, Excel 업로드, 예시 케이스 리셋
  • General Settings: 전체 설정값을 테이블 형태로 일괄 편집
  • Sales Orders: 주문 납기/물량/spec 수정
  • Blendstocks: 반제품 성상/원가/재고/rundown 수정
  • Product Tanks: 탱크 용량/초기 volume/초기 성상 수정
  • Notes: 기본 예시와 모델 개요 설명

6.3 Results 탭

  • Campaign Gantt: 블렌더 라인별 배치 실행 구간 표시
  • Batch Detail: 선택 배치의 Batch Info / Blend Recipe / Quality vs Target 표시
  • Tank Level: 제품 탱크 수위 추이
  • Inventory: 반제품 재고 추이
  • Summaries: 주문/슬랙/반제품 사용량 및 runtime profile 요약

6.4 Batch Detail 해석

  • Batch Info: 제품, 주문, 탱크, 단계, 볼륨, 시간, due 정보
  • Blend Recipe: 배치 성분 투입량과 HEEL 물량 표시
  • Quality vs Target: Heel / Blend / Target 비교
  • 최종 배치에서 spec 위반 시 빨간색, 경계 1% 이내면 노란색으로 강조

7. Excel 입력 템플릿

8. 성능 기준

현재 2개월 기본 예시 기준 검증 결과는 다음과 같습니다.

  • 총 주문: 70건
  • 총 배치: 420개
  • 총 풀이 시간: 약 30초 수준
  • Inventory slack: 0 수준으로 관리

9. 실행 방법

  1. 프로젝트 폴더로 이동
  2. pip install -r requirements.txt
  3. python3 -m streamlit run app.py
  4. 브라우저에서 http://localhost:8501 접속

10. 현재 한계와 향후 개선 방향

  • 전역 MINLP가 아니라 휴리스틱 기반 구조
  • overflow를 더 적극적으로 줄이는 inventory overflow repair는 추가 여지 있음
  • berth/vessel scheduling, tank cleaning, scenario library는 아직 미구현
  • 실제 정유사 master data와 연결하면 실무 적용성이 더 높아질 수 있음
